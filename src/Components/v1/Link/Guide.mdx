# 🧭  Guia de Uso

## 📘 Visão Geral

O componente `Link` é utilizado para criar links de navegação baseados na tag `<a>`, permitindo que os usuários possam ser redirecionados para outras páginas ou seções. O `Link` foi projetado desde o início para ser acessível e intuitivo, seguindo as diretrizes de usabilidade e acessibilidade recomendadas pelas <Link className="sb-unstyled text-sm" href="https://www.w3.org/WAI/standards-guidelines/wcag/" target="_blank">Diretrizes WCAG</Link>.

Por meio das suas variações de estados, o `Link` assegura uma experiência de navegação consistente e clara, facilitando a compreensão do propósito de cada link e como interagir com ele, além de garantir que todos os links sejam facilmente identificáveis e acessíveis.

<img src="/assets/link-variants.png" alt="Exibição do componente de Link em uma lista com as suas variações e estados como hover, focus e disabled e aplicação com ícone" style={{ margin: '0 auto' }} />

Um componente `Link` é indicado para criar links de navegação para:

- Uma nova URL;
- Uma seção específica da mesma página;
- Um documento ou recurso externo;

## 💡 Recomendações de Uso

- Utilize links para navegação entre páginas ou seções, evitando seu uso para ações que não envolvam redirecionamento;
- Os textos dos links devem ser claros e descritivos, indicando exatamente o que acontecerá ao serem ativados;
- Evite usar links para "envolver" outros componentes de ação, como botões, no código HTML, pois isso pode comprometer a acessibilidade e a navegação;
- Mantenha o padrão de abertura dos links na mesma janela, deixando o usuário livre para abrir em nova aba quando desejar;
- Utilize `target="_blank"` para abrir links em nova aba ou janela somente quando necessário, como em links externos, documentos fora da navegação principal ou quando for importante que a página atual permaneça aberta;
- Preserve os estilos visuais padrão dos links, como sublinhado e seus estados, para garantir consistência e facilitar a identificação;
- Embora a propriedade `disabled` possa ser usada para indicar que o link está indisponível, evite links desabilitados, pois podem causar confusão ao usuário;
Utilize o componente `Button` para ações que executam uma operação na própria página, como enviar formulários, abrir modais, toggles, ou executar comandos sem navegação;
- Não remova o sublinhado dos links, pois isso pode dificultar a identificação de que se trata de um link clicável;
- Caso haja múltiplos elementos interativos próximos, recomenda-se manter um espaçamento apropriado entre eles para evitar ativações não intencionais, especialmente em dispositivos sensíveis ao toque.

## ♿ Acessibilidade

O componente `Link` foi desenvolvido com foco em acessibilidade e também testado para atender às <Link className="sb-unstyled text-sm" href="https://www.w3.org/WAI/standards-guidelines/wcag/" target="_blank">Diretrizes WCAG</Link>, garantindo que todos os usuários possam interagir com ele de forma eficaz. Assegurando que:

- O link pode ser ativado pela tecla Enter no teclado, conforme o comportamento padrão de elementos `<a>` nativos, alinhado às diretrizes da <Link className="sb-unstyled text-sm" href="https://www.w3.org/WAI/WCAG21/Understanding/keyboard.html" target="_blank">WCAG 2.1.1 (Keyboard)</Link>;
- A borda de foco visível possui pelo menos **2 píxeis** de espessura e está sempre visível, sem ficar oculta por outros elementos, conforme <Link className="sb-unstyled text-sm" href="https://www.w3.org/WAI/WCAG22/Understanding/focus-appearance-enhanced" target="_blank">WCAG 2.4.13 (Focus Appearance - Enhanced)</Link>;
- O estado de foco é sempre preservado quando aplicável, incluindo um suporte visual claro,, conforme recomendado pela <Link className="sb-unstyled text-sm" href="https://www.w3.org/WAI/WCAG21/Understanding/focus-visible" target="_blank">WCAG 2.4.7 (Focus Visible)</Link>. Mesmo quando o link está desabilitado, o estilo visual de foco é mantido para reforçar a percepção de que ele está presente na interface, promovendo melhor experiência para os usuários.

### 📌 Boas práticas de acessibilidade ao utilizar o componente

Para complementar e garantir a acessibilidade, recomenda-se:

- O texto do link deve ser claro e descritivo para que seu propósito seja compreendido pelo usuário, conforme a <Link className="sb-unstyled text-sm" href="https://www.w3.org/WAI/WCAG21/Understanding/link-purpose-in-context.html" target="_blank">WCAG 2.4.4 (Link Purpose in Context)</Link>;
- Quando não for possível usar um texto suficientemente descritivo, utilize o atributo `aria-label` para fornecer uma descrição acessível para leitores de tela, conforme recomendado na <Link className="sb-unstyled text-sm" href="https://www.w3.org/WAI/WCAG21/Techniques/aria/ARIA10.html" target="_blank">WCAG ARIA10</Link>;
- Quando os links estão desativados na interface do usuário, as diretrizes de contraste de cores não se aplicam a esses componentes. As <Link className="sb-unstyled text-sm" href="https://www.w3.org/WAI/WCAG21/Understanding/contrast-minimum.html#exceptions" target="_blank">diretrizes de contraste mínimo definidas nas WCAG 2.1</Link> declaram explicitamente que elementos inativos estão isentos da exigência;
- O componente já possui as propridades `aria-disabled` e `data-disabled` para indicar que o link está desativado, mas é importante se atentar ao uso da propridade `disabled` quando o botão não deve ser clicável e garantir que o estado visual também seja claro;
- É recomendado informar o motivo do link estar desabilitado para garantir clareza ao usuário, seguindo as diretrizes de <Link className="sb-unstyled text-sm" href="https://www.w3.org/WAI/WCAG21/Understanding/info-and-relationships.html" target="_blank">WCAG 1.3.1 (Information and Relationships)</Link> e <Link className="sb-unstyled text-sm" href="https://www.w3.org/WAI/WCAG21/Understanding/labels-or-instructions.html" target="_blank">WCAG 3.3.2 (Labels or Instructions)</Link>, podendo usar texto visível ou `aria-describedby`;
- Preserve os recursos visuais de acessibilidade (foco, contraste) ao customizar estilos ou tokens, mantendo a consistência com os testes de acessibilidade já realizados; conforme <Link className="sb-unstyled text-sm" href="https://www.w3.org/WAI/WCAG21/Understanding/focus-visible.html" target="_blank">WCAG 2.4.7 (Focus Visible)</Link>, <Link className="sb-unstyled text-sm" href="https://www.w3.org/WAI/WCAG21/Understanding/contrast-minimum.html" target="_blank">WCAG 1.4.3 (Contrast Minimum)</Link> e <Link className="sb-unstyled text-sm" href="https://www.w3.org/WAI/WCAG22/Understanding/focus-appearance-enhanced.html" target="_blank">WCAG 2.4.13 (Focus Appearance - Enhanced)</Link>.
