# 🧭  Guia de Uso

## 📘 Visão Geral

O componente `Grid` permite a criação de *layouts* baseados em grade, organizando elementos em linhas e colunas. Ele utiliza o modelo de *layout* CSS Grid, oferecendo flexibilidade para definir o número de colunas, linhas, espaçamentos e alinhamentos dos itens por meio de suas propriedades baseadas em classes do Tailwind CSS.

## 💡 Recomendações de Uso

- Utilize o componente `Grid` para criar *layouts* baseados em grade, organizando elementos de forma estruturada em linhas e colunas;
- Aproveite as propriedades de colunas (`cols`), linhas (`rows`), alinhamento (`align`) e justificação (`justify`) para controlar a disposição dos elementos dentro da grade;
- Utilize as propriedades `rowGap` e `colGap` para definir o espaçamento entre as linhas e colunas, melhorando a legibilidade e a estética do *layout*;	
- Teste o *layout* em diferentes tamanhos de tela e resoluções para garantir uma organização responsiva e consistente. Considere também o uso de *breakpoints* para adaptar a estrutura a diferentes dispositivos e larguras de tela.

## ♿ Acessibilidade

O componente `Grid` foi desenvolvido e testado para atender às <Link className="sb-unstyled text-sm" href="https://www.w3.org/WAI/standards-guidelines/wcag/" target="_blank">Diretrizes WCAG</Link>, oferecendo suporte a propriedades de acessibilidade e práticas recomendadas para um conteúdo acessível. Para garantir que o uso do componente `Grid` esteja em conformidade com as diretrizes de acessibilidade, é importante seguir algumas práticas recomendadas:

- Use o componente `Grid` para organizar elementos de forma que a ordem visual reflita a ordem lógica do conteúdo no HTML, garantindo que leitores de tela interpretem corretamente a estrutura da página. Evite layouts com leitura não linear causados por grid-template-areas, order ou grid-auto-flow: dense, que podem criar uma ordem visual que não corresponde à ordem do DOM. Isso está alinhado com a <Link className="sb-unstyled text-sm" href="https://www.w3.org/WAI/WCAG21/Understanding/meaningful-sequence.html" target="_blank">WCAG 1.3.2 (Meaningful Sequence)</Link>;
- Evite utilizar order ou manipulação de grid para alterar visualmente a ordem dos elementos sem refletir essa alteração no HTML. Tecnologias assistivas leem o conteúdo na ordem do DOM, o que pode causar confusão se a ordem visual e a ordem de leitura forem diferentes. Essa prática também fere a <Link className="sb-unstyled text-sm" href="https://www.w3.org/WAI/WCAG21/Understanding/info-and-relationships.html" target="_blank">WCAG 1.3.1 (Info and Relationships)</Link>;
- Evite o uso excessivo de elementos `div` ou `span` sem propósito semântico. Sempre que possível, utilize a propriedade `component` para alterar o elemento renderizado e incluir tags HTML semânticas apropriadas (como `header`, `nav`, `main`, `section`, `article`, etc.), melhorando a compreensão por tecnologias assistivas, conforme a <Link className="sb-unstyled text-sm" href="https://www.w3.org/WAI/WCAG21/Understanding/info-and-relationships.html" target="_blank">WCAG 1.3.1 (Info and Relationships)</Link>;
- Utilize classes de acessibilidade do Tailwind CSS, como `sr-only`, para ocultar visualmente conteúdos que devem permanecer acessíveis a leitores de tela, garantindo a entrega de informações completas a todos os usuários, conforme a <Link className="sb-unstyled text-sm" href="https://www.w3.org/WAI/WCAG21/Understanding/non-text-content.html" target="_blank">WCAG 1.1.1 (Non-text Content)</Link>;
- Certifique-se de que os elementos dentro do `Grid` sigam uma ordem de tabulação lógica e previsível ao navegar com o teclado, respeitando a sequência visual e funcional do conteúdo, conforme a <Link className="sb-unstyled text-sm" href="https://www.w3.org/WAI/WCAG21/Understanding/focus-order.html" target="_blank">WCAG 2.4.3 (Focus Order)</Link>.
