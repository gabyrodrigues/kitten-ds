# 🧭  Guia de Uso

## 📘 Visão Geral

O componente `CheckboxGroup` é utilizado para agrupar múltiplos componentes `Checkbox` (caixas de seleção) em um único contexto, permitindo que o usuário selecione várias opções simultaneamente. Ele é ideal para formulários, questionários e configurações de preferências, onde o usuário pode escolher diversas alternativas dentro de um mesmo conjunto.

### ⚙️ Tipos de CheckboxGroup

<img src="/assets/checkbox-group-variants.png" alt="Exibição do componente de CheckboxGroup em uma lista com suas variantes de orientação horizontal e vertical" style={{ margin: '0 auto' }} />

O componente `CheckboxGroup` oferece duas orientações principais:

- **Horizontal**: As opções são dispostas em uma linha, uma ao lado da outra. Útil para opções mais curtas ou quando o espaço horizontal é suficiente, proporcionando uma visualização compacta e rápida das opções disponíveis;
- **Vertical**: As opções são dispostas em uma coluna, uma abaixo da outra. Orientação padrão do `CheckboxGroup`, facilitando a leitura e a seleção de opções em listas mais longas.

O `CheckboxGroup` também oferece suporte à obrigatoriedade. Através da combinação das propriedades `withAsterisk` e `required`, é possível indicar visualmente (com um asterisco (*)) que pelo menos uma opção precisa ser selecionada. Isso é útil em formulários onde a seleção de pelo menos uma opção é necessária para o envio.

Como o `CheckboxGroup` é um agrupamento para múltiplos componentes `Checkbox`, ele não possui variantes de cor ou estado próprios, mas sim herda as características dos `Checkbox` que contém. Consulte as variantes, estados e outras orientações do `Checkbox` no guia de uso do componente [**Checkbox**](/docs/componentes-checkbox-guia-de-uso--propriedades).


## 🖥️ Exemplo de Uso

> O componente `CheckboxGroup` está relacionado ao componente [**Checkbox**](/docs/componentes-checkbox-guia-de-uso--propriedades), que representa cada opção individual dentro do grupo. O uso do `CheckboxGroup` exige que os elementos `Checkbox` estejam encapsulados em seu interior para que funcionem corretamente.

No exemplo abaixo, é possível ver como utilizar o `CheckboxGroup` com múltiplos `Checkbox`, organizando visualmente as opções e permitindo múltiplas seleções. Além disso, é possível observar como o `CheckboxGroup` pode repassar propriedades como `color` e `disabled` para os seus `Checkbox` filhos:

```javascript
import { Checkbox, CheckboxGroup } from "kitten-ds"
function App() {
  return (
    <div>
      <CheckboxGroup
        label="Selecione suas comidas favoritas"
        color="primary"
        disabled
      >
        <Checkbox
          label="Bacon"
          value="bacon"
        />
        <Checkbox
          label="Cupcake"
          value="cupcake"
        />
        <Checkbox
          label="Pizza"
          value="pizza"
        />
        <Checkbox
          label="Sushi"
          value="sushi"
        />
      </CheckboxGroup>
    </div>
  );
}

export default App;
```

> 🎨 A propriedade `color` definida no `CheckboxGroup` será aplicada automaticamente a todos os `Checkbox` dentro do grupo, garantindo consistência visual. O mesmo vale para `disabled`, que desabilita todos os itens do grupo quando ativada.

> ℹ️ Para mais detalhes sobre todas as propriedades disponíveis no `CheckboxGroup` e como elas se relacionam com os componentes `Checkbox`, consulte a [documentação de propriedades do CheckboxGroup](/docs/componentes-checkboxgroup--propriedades).

## 💡 Recomendações de Uso

- Prefira o uso de `CheckboxGroup` quando houver um número limitado de opções (geralmente até 10) e for necessário permitir múltiplas seleções;
- Utilize o `CheckboxGroup` em situações onde as opções não são mutuamente exclusivas, ou seja, a seleção de uma opção não impede a seleção de outra;
- Organize as opções de `Checkbox` de forma lógica, como por exemplo em ordem alfabética, por prioridade, frequência de uso ou relevância para o usuário;
- Mantenha a consistência visual com o restante da interface, utilizando variantes de cor apropriadas para indicar o propósito ou prioridade da escolha;
- Forneça *feedback* visual claro para os diferentes estados do `Checkbox`, como erro, sucesso ou ajuda;
- Utilize rótulos claros em cada opção de `Checkbox`, além de também utilizar um rótulo descritivo para o `CheckboxGroup`, que pode indicar a categoria do agrupamento ou descrever o propósito do grupo de opções;
- Evite deixar opções marcadas por padrão, pois podem passar despercebidas e resultar em respostas não intencionais;
- Alinhe a lista verticalmente sempre que possível e use a orientação horizontal apenas quando as opções forem curtas e o espaço permitir, para evitar que o usuário precise rolar horizontalmente.

## ♿ Acessibilidade

O componente `CheckboxGroup`, por agrupar componentes `Checkbox`, segue as mesmas diretrizes e recomendações aplicadas ao `Checkbox`. Além disso, o `CheckboxGroup` também possui características adicionais importantes:

- Mensagens de sucesso, ajuda ou erro são associadas ao grupo por meio do atributo `aria-describedby`, permitindo que leitores de tela as anunciem corretamente, conforme a <Link className="sb-unstyled text-sm" href="https://www.w3.org/WAI/WCAG21/Understanding/info-and-relationships.html" target="_blank">WCAG 1.3.1 (Info and Relationships)</Link>;
- Para mensagens dinâmicas (erro ou sucesso) do grupo, é utilizado o atributo `aria-live="polite"`, garantindo que leitores de tela anunciem essas mudanças de forma não intrusiva, conforme a <Link className="sb-unstyled text-sm" href="https://www.w3.org/WAI/WCAG21/Understanding/status-messages.html" target="_blank">WCAG 4.1.3 (Status Messages)</Link>.

### ⌨️ Navegação por teclado

O `CheckboxGroup` é totalmente **acessível via teclado**, conforme recomendado pela <Link className="sb-unstyled text-sm" href="https://www.w3.org/WAI/WCAG21/Understanding/keyboard.html" target="_blank">WCAG 2.1.1 (Keyboard)</Link>. A navegação do componente funciona da seguinte forma:

- **Tab**: Move o foco para o próximo `Checkbox` do grupo;
- **Shift + Tab**: Move o foco para o `Checkbox` anterior do grupo;
- **Espaço**: Alterna o estado (marcado ou desmarcado) do `Checkbox` com foco.

### 📌 Boas práticas de acessibilidade ao utilizar o componente

Além das diretrizes já implementadas no `CheckboxGroup` e no `Checkbox`, considere as seguintes práticas ao utilizar o `CheckboxGroup`:

- Sempre forneça um **rótulo claro e descritivo** para o grupo por meio da propriedade `label`, indicando a categoria ou propósito do grupo de opções, conforme a <Link className="sb-unstyled text-sm" href="https://www.w3.org/WAI/WCAG21/Understanding/labels-or-instructions.html" target="_blank">WCAG 3.3.2 (Labels or Instructions)</Link>;
- Se for **necessário** que o `CheckboxGroup` **não possua um rótulo visível**, o componente **aplica automaticamente um rótulo acessível** padrão através da propriedade `defaultA11yLabel`, que pode ser **customizada** conforme a necessidade para garantir que leitores de tela compreendam o propósito do grupo. Essa prática está alinhada com a <Link className="sb-unstyled text-sm" href="https://www.w3.org/WAI/WCAG22/Understanding/headings-and-labels.html" target="_blank">WCAG 2.4.6 (Headings and Labels)</Link>;
- Sempre que possível, realize a **validação da opção selecionada nos grupos antes do envio do formulário**, exibindo mensagens de erro claras e específicas, conforme a <Link className="sb-unstyled text-sm" href="https://www.w3.org/WAI/WCAG21/Understanding/error-identification.html" target="_blank">WCAG 3.3.1 (Error Identification)</Link>.
