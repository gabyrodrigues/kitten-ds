# 🧭  Guia de Uso

## 📘 Visão Geral

O componente `Title` é utilizado para criar títulos e cabeçalhos. Ele oferece variantes baseadas nas tags de título HTML (`h1`, `h2`, etc.) e variantes de exibição (`display1`, `display2`, etc.), cada uma projetada para se adequar a diferentes casos de uso e tipos de conteúdo. O componente `Title` foi desenvolvido a partir de classes do Tailwind CSS e são totalmente customizáveis, permitindo que você escolha a variante mais adequada dependendo do contexto do conteúdo.

O componente `Title` pode ser utilizado dessa forma:

```tsx
<Title variant="h1">H1 Variant Title</Title>
```

## ⚙️ Propriedades

O componente `Title` possui as propriedades `variant` e `component`, que permitem definir a variante do título e o elemento HTML a ser utilizado, respectivamente. O componente também aceita propriedades como `fontSize`, `weight`, `lineHeight`, `textAlign`, `color`, e `className` para personalização adicional, todas baseadas nas classes do Tailwind CSS.

A propriedade `variant` aceita valores como `display1`, `display2`, `display3`, `h1`, `h2`, `h3`, que possuem tamanhos e estilos específicos, como `text-display1`, `text-display2`, `text-display3`, `text-h1`, `text-h2`, `text-h3`, `text-h4`, `text-h5`, `text-h6`. Essas variantes são projetadas para diferentes tamanhos e pesos de fonte, adequando-se a títulos de diferentes níveis e importâncias. Por exemplo, as variantes de exibição são geralmente usadas para títulos maiores e mais destacados, enquanto as variantes de cabeçalho (`h1`, `h2`, etc.) são usadas para títulos hierárquicos em documentos e páginas.

A propriedade `component` permite a escolha do elemento HTML a ser usado para o título, como `h1`, `h2`, `h3`, etc. Isso é útil para garantir que o título seja semanticamente correto e acessível, além de permitir que você use o componente em diferentes contextos, como títulos de seções, subtítulos, etc.

## 💡 Recomendações de Uso

- Utilize o componente `Title` para criar títulos e cabeçalhos que sejam semanticamente corretos e acessíveis;
- Escolha a variante adequada (`display1`, `display2`, `h1`, etc.) com base no contexto do conteúdo e na hierarquia visual desejada;
- Use a propriedade `component` para garantir que o título seja renderizado com o elemento HTML correto, como `h1`, `h2`, etc., sempre verificando a ordem correta de hierarquia de títulos para garantir a acessibilidade e a estrutura do conteúdo;
- Dê preferência ao uso da propriedade `variant` para definir os estilos do título, mantendo a consistência e a semântica do componente, porém, as propriedades `fontSize`, `weight`, `lineHeight`, `textAlign`, `color` e `className` também podem ser utilizadas para personalização adicional, se necessário;
- Evite o uso excessivo de títulos de exibição (`display1`, `display2`) em contextos onde títulos hierárquicos são mais apropriados, como em documentos ou páginas com estrutura clara.

## ♿ Acessibilidade

O componente `Title` foi desenvolvido e testado para atender às <Link className="sb-unstyled text-sm" href="https://www.w3.org/WAI/standards-guidelines/wcag/" target="_blank">Diretrizes WCAG</Link>, oferecendo suporte a propriedades de acessibilidade e práticas recomendadas para um conteúdo acessível. Para garantir que o uso do componente `Title` esteja em conformidade com as diretrizes de acessibilidade, é importante seguir algumas práticas recomendadas:

- Utilize o componente `Title` para criar títulos e cabeçalhos que sejam semanticamente corretos, garantindo que a hierarquia do conteúdo seja clara e compreensível por leitores de tela, conforme a <Link className="sb-unstyled text-sm" href="https://www.w3.org/WAI/WCAG21/Understanding/info-and-relationships.html" target="_blank">WCAG 1.3.1 (Info and Relationships)</Link>;
- Escolha a variante adequada (`display1`, `display2`, `h1`, etc.) de acordo com a hierarquia visual e semântica desejada. Sempre que necessário, utilize a propriedade `component` para renderizar o título com a tag HTML correta (`h1`, `h2`, etc.), respeitando a estrutura lógica do conteúdo, conforme a <Link className="sb-unstyled text-sm" href="https://www.w3.org/WAI/WCAG21/Understanding/info-and-relationships.html" target="_blank">WCAG 1.3.1 (Info and Relationships)</Link>;
- Use títulos descritivos que expressem com clareza o conteúdo ou propósito da seção, conforme a <Link className="sb-unstyled text-sm" href="https://www.w3.org/WAI/WCAG21/Understanding/headings-and-labels.html" target="_blank">WCAG 2.4.6 (Headings and Labels)</Link>;
- Garanta que o contraste entre o texto do título e o fundo esteja de acordo com os critérios de contraste mínimo, conforme a <Link className="sb-unstyled text-sm" href="https://www.w3.org/WAI/WCAG21/Understanding/contrast-minimum.html" target="_blank">WCAG 1.4.3 (Contrast - Minimum)</Link>. Dê preferência aos *tokens* de tipografia disponíveis, como `text-typography-primary`, `text-typography-secondary`, etc., para manter consistência visual e boa legibilidade.
