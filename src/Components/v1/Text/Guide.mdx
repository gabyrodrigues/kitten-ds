# 🧭  Guia de Uso

## 📘 Visão Geral

O componente `Text`, diferente do componente `Title`, é utilizado para exibir textos comuns, como parágrafos, legendas e outros conteúdos textuais que não são títulos ou cabeçalhos. Ele é projetado para ser flexível e adaptável a diferentes contextos de uso, oferecendo várias variantes de estilo baseadas nas classes do Tailwind CSS.

O componente `Text` pode ser utilizado dessa forma:

```tsx
<Text variant="body1">Body 1 Variant Text</Text>
```

### ⚙️ Propriedades

O componente `Text` possui as propriedades `variant` e `component`, que permitem definir a variante do título e o elemento HTML a ser utilizado, respectivamente. O componente também aceita propriedades como `fontSize`, `weight`, `lineHeight`, `textAlign`, `color`, e `className` para personalização adicional, todas baseadas nas classes do Tailwind CSS.

A propriedade `variant` aceita valores como `body1`, `body2`, `body3`, `body4`, que correspondem a diferentes estilos de texto, como tamanhos de fonte, pesos e espaçamento entre letras. Essas variantes são projetadas para atender a diferentes necessidades de exibição de texto, desde parágrafos regulares até textos menores e mais compactos.
A propriedade `component` permite escolher o elemento HTML a ser usado para o texto, como `p`, `span`, `div`, etc. Isso é útil para garantir que o texto seja semanticamente correto e acessível, além de permitir que o componente seja utilizado em diferentes contextos, como parágrafos, legendas, etc.

## 💡 Recomendações de Uso

- Utilize o componente `Text` para exibir textos comuns, como parágrafos, legendas e outros conteúdos textuais que não são títulos ou cabeçalhos;
- Escolha a variante adequada (`body1`, `body2`, `body3`, `body4`) com base no tamanho e estilo do texto desejado, garantindo que o conteúdo seja legível e visualmente consistente;
- Use a propriedade `component` para garantir que o texto seja renderizado com o elemento HTML correto, como `p`, `span`, etc., sempre verificando a semântica do conteúdo e a acessibilidade;
- Dê preferência ao uso da propriedade `variant` para definir os estilos do texto, mantendo a consistência e a semântica do componente, porém, as propriedades `fontSize`, `weight`, `lineHeight`, `textAlign`, `color` e `className` também podem ser utilizadas para personalização adicional, se necessário;
- Evite o uso excessivo de estilos personalizados que possam comprometer a legibilidade ou a hierarquia do conteúdo, preferindo as variantes predefinidas para manter a consistência visual.

## ♿ Acessibilidade

O componente `Text` foi desenvolvido e testado para atender às <Link className="sb-unstyled text-sm" href="https://www.w3.org/WAI/standards-guidelines/wcag/" target="_blank">Diretrizes WCAG</Link>, oferecendo suporte a propriedades de acessibilidade e práticas recomendadas para um conteúdo textual acessível. Para garantir que o uso do componente `Text` esteja em conformidade com essas diretrizes, siga as recomendações abaixo:

- Utilize o componente `Text` para exibir textos comuns, como parágrafos, legendas e outros conteúdos textuais que não são títulos ou cabeçalhos. Isso assegura que o conteúdo seja lido de forma clara por tecnologias assistivas, conforme a <Link className="sb-unstyled text-sm" href="https://www.w3.org/WAI/WCAG21/Understanding/info-and-relationships.html" target="_blank">WCAG 1.3.1 (Info and Relationships)</Link>;
- Escolha a variante adequada (`body1`, `body2`, `body3`, `body4`) com base no tamanho e estilo do texto desejado, promovendo a legibilidade e a consistência visual. O uso apropriado de estilos tipográficos contribui para a acessibilidade do conteúdo, conforme a <Link className="sb-unstyled text-sm" href="https://www.w3.org/WAI/WCAG21/Understanding/visual-presentation.html" target="_blank">WCAG 1.4.8 (Visual Presentation)</Link>;
- Use a propriedade `component` para garantir que o texto seja renderizado com o elemento HTML apropriado, como `p`, `span`, etc. Isso melhora a interpretação da estrutura do conteúdo por leitores de tela, conforme a <Link className="sb-unstyled text-sm" href="https://www.w3.org/WAI/WCAG21/Understanding/info-and-relationships.html" target="_blank">WCAG 1.3.1 (Info and Relationships)</Link>;
- Dê preferência à propriedade `variant` para manter estilos tipográficos padronizados. Personalizações adicionais com `fontSize`, `weight`, `lineHeight`, `textAlign`, `color` e `className` devem preservar a clareza visual e facilitar a leitura do conteúdo, conforme a <Link className="sb-unstyled text-sm" href="https://www.w3.org/WAI/WCAG21/Understanding/visual-presentation.html" target="_blank">WCAG 1.4.8 (Visual Presentation)</Link>;
- Garanta que o contraste entre o texto e o fundo esteja dentro dos critérios mínimos exigidos, conforme a <Link className="sb-unstyled text-sm" href="https://www.w3.org/WAI/WCAG21/Understanding/contrast-minimum.html" target="_blank">WCAG 1.4.3 (Contrast - Minimum)</Link>. Utilize *tokens* de cor pré-definidos, como `text-typography-primary`, `text-typography-secondary`, etc., para manter boa legibilidade e consistência.
