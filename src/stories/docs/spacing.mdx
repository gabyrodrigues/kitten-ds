# 📏 Espaçamento do Kitten DS

O sistema de espaçamento do **Kitten DS** é baseado em uma escala modular que segue incrementos de `4px`, alinhada à abordagem do Tailwind CSS. Essa escala padronizada garante consistência visual, facilita o alinhamento dos componentes e cria padrões de espaçamento previsíveis em toda a interface.

O **Kitten DS** utiliza essa escala para definir *tokens* de espaçamento, que podem ser aplicados em margens, preenchimentos e espaçamentos entre elementos. Esses *tokens* são fundamentais para manter a consistência no desenvolvimento de componentes e *layouts*, permitindo que eles possuam um padrão de espaçamento uniforme.

> 💡 **Por que isso importa?**  
> Ao adotar uma escala de espaçamento consistente por meio do uso de *tokens*, é possível reduzir erros de alinhamento, melhorar a legibilidade e simplificar a manutenção. Alterações futuras tornam-se mais rápidas e seguras, já que os espaçamentos ficam concentrados na configuração do tema, evitando ajustes manuais e margens avulsas.

## ℹ️ Uso recomendado dos tokens de espaçamento

- **`gap`**: Usado em containers (`Flex`, `Grid`) para definir espaçamento entre elementos irmãos, como campos de formulários, listas, blocos ou botões alinhados;
- **`padding` (`p-`, `px-`, `py-`)**: Usado para espaçamento interno, por exemplo, no conteúdo dentro de botões, inputs, cards e containers;
- **`margin` (`m-`, `mx-`, `my-`)**: Ideal para espaçamento externo, separando títulos das seções ou labels dos inputs.

Essa divisão ajuda a manter o código limpo, evita regras sobrepostas e facilita ajustes futuros para manter a consistência do design.

> ✅ **Dica prática:**  
> Usar `gap` como padrão simplifica muito a construção dos *layouts*. Isso é especialmente relevante porque muitos componentes do *design system*, como `Input` e `Button`, já possuem espaçamentos internos (via `padding`) e não exigem margens adicionais. Dessa forma, o container que agrupa esses elementos é responsável pelo espaçamento entre eles.

> ⚠️ **Atenção:**  
> No **Kitten DS**, o uso de `gap` é priorizado nos containers (`Flex`, `Grid`) para definir o espaçamento entre elementos que fazem parte de uma mesma estrutura (irmãos dentro do mesmo container). Já o uso de `margin` é recomendado quando:
> - Não existe um container controlando o espaçamento, como na separação entre um título e um bloco de conteúdo, ou entre seções da página. Para elementos como `Input` e `Button`, o próprio componente já encapsula os espaçamentos internos, incluindo `labels` quando aplicável;
> - Ou quando você deseja criar um espaçamento externo em relação a outros blocos.

> 🚫 Evite misturar `margin` e `gap` no mesmo grupo de elementos, pois isso gera espaçamentos duplicados e desalinhamentos.

## 🛠️ Exemplo prático de espaçamento com Kitten DS

Aqui temos um *layout* simples utilizando os componentes `Flex`, `Title`, `Input` e `Button`, aplicando os espaçamentos recomendados segundo este guia:

```jsx live

export function SpacingExample() {
  return (
    <Flex direction="flex-col" paddingX="px-xl" paddingY="py-xl">
      <Title variant="h4" marginBottom="mb-lg">
        Formulário de Contato
      </Title>

      {/* Container principal com espaçamento entre os blocos */}
      <Flex direction="flex-col" gap="gap-lg" className="sb-unstyled">
        {/* Campos do formulário, espaçamento vertical entre inputs via gap */}
        <Flex direction="flex-col" gap="gap-sm">
          <Input placeholder="Digite seu nome" full />
          <Input placeholder="Digite seu email" full />
        </Flex>

        {/* Botões alinhados horizontalmente com gap */}
        <Flex direction="flex-row" colGap="gap-x-sm" width="w-full">
          <Button color="primary" full>Enviar</Button>
          <Button color="error" full>Cancelar</Button>
        </Flex>
      </Flex>
    </Flex>
  );
}
```

> ✨ Nota: A utilização correta dos *tokens* de espaçamento não é apenas uma questão estética, mas também contribui para interfaces mais legíveis, acessíveis e com maior coerência visual. O **Kitten DS**, alinhado ao Tailwind, promove essa consistência de forma simples e escalável.
> **Seguir essas recomendações reduz retrabalho, evita desalinhamentos e torna o código mais fácil de manter.**
